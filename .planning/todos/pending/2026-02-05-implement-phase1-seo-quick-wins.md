## Problem

Public pages audit (docs/public-pages-audit.md) identified critical SEO gaps that are quick wins:

1. **Missing metadata** — Landing page (`/`), browse page (`/app`), and jobs page (`/jobs`) use only global metadata from layout.tsx, missing page-specific titles and descriptions
2. **No sitemap.xml** — Search engines can't efficiently discover and index all pages
3. **No robots.txt** — No crawl directives to protect admin routes or guide indexing
4. **Missing FAQ sections** — Landing page lacks FAQ schema for featured snippets
5. **No H1 tags** — Browse page "Browse all vendors" should be semantic H1

These are foundational SEO requirements that block effective indexing and ranking.

## Solution

**Phase 1 Quick Wins (1-2 weeks estimated):**

### 1. Add page-specific metadata

**Landing page (src/app/page.tsx):**
```typescript
export const metadata: Metadata = {
  title: 'Melbourne Mobile Coffee Carts | The Bean Route',
  description: 'Book verified mobile coffee carts in Melbourne. Serving CBD, Carlton, Fitzroy & 20+ suburbs. Corporate events, weddings, festivals. Free quotes in 60 seconds.',
}
```

**Browse page (src/app/app/page.tsx):**
```typescript
export const metadata: Metadata = {
  title: 'Browse Melbourne Coffee Cart Vendors | The Bean Route',
  description: 'Compare 10+ verified coffee cart vendors across Melbourne. Filter by suburb, event type, and price. Free quotes from trusted mobile baristas.',
}
```

**Jobs page (src/app/jobs/page.tsx):**
```typescript
export const metadata: Metadata = {
  title: 'Melbourne Coffee Cart Jobs | The Bean Route',
  description: 'Find mobile coffee cart opportunities in Melbourne. Event organizers post jobs, vendors submit quotes. Free to join.',
}
```

### 2. Generate sitemap.xml

Use `next-sitemap` package:
```bash
npm install next-sitemap --save-dev
```

Create `next-sitemap.config.js`:
```js
module.exports = {
  siteUrl: 'https://thebeanroute.com.au',
  generateRobotsTxt: true,
  exclude: ['/admin', '/admin/*', '/design-system'],
}
```

Add to package.json:
```json
"scripts": {
  "postbuild": "next-sitemap"
}
```

### 3. Create robots.txt

If not auto-generated by next-sitemap, create `public/robots.txt`:
```
User-agent: *
Allow: /
Disallow: /admin
Disallow: /api/
Disallow: /design-system

Sitemap: https://thebeanroute.com.au/sitemap.xml
```

### 4. Add FAQ section to landing page

Add before final CTA section in src/app/page.tsx:
- "How much does a mobile coffee cart cost in Melbourne?"
- "Which Melbourne suburbs do you cover?"
- "How far in advance should I book?"
- "Do coffee carts need power at my event?"

Include FAQ schema using existing JsonLd component.

### 5. Fix H1 tags

Browse page: Change "Browse all vendors" to semantic `<h1>` tag.

**Expected impact:**
- Better indexing by search engines
- Improved click-through rates from search results
- Foundation for featured snippets
- Proper crawl budget allocation

**Dependencies:**
- None — can implement immediately
- Builds on existing JsonLd component and metadata system

**Testing:**
- Verify sitemap.xml accessible at https://thebeanroute.com.au/sitemap.xml
- Use Google Rich Results Test for FAQ schema
- Check metadata in browser dev tools
- Submit sitemap to Google Search Console after deployment
